Intersection Testing

Justification section.

Intersection Shapes in 2D

Note: "Line" includes line segments as well as rays. Polynomial (n) stands for a polynomial of degree n.

We will use this short naming convention to fit all the data into a small table: Pnt: point, Ln: line, Crc: circle, Pln: polynomial.

Coefficients in front of an object counts how many there can be for one case of the intersection test. Programmatically, we are always dealing with floating-point errors, and certain mathematical possibilities will only rarely happen in programming. We mark these rare cases with an asterisk (*).

There is always the possibility of no intersection happening between two mathematical objects. In this case, we will omit writing null, but always keep in mind that this might be the case for your intersection tests as well.

Some objects, like functions and curves, are too vague to discern certain intersection properties with other objects. We leave these empty, knowing that we have to specify further.

Point Line Circle Polynomial (n)
Point 1Pnt 1Pnt 1Pnt nPnt, (n-1)Pnt, ..., 1Pnt
Line 1Pnt 1Pnt 2Pnt, 1Pnt*
Circle 1Pnt 1Pnt Crc*, 2Pnt, 1Pnt* 1Pnt
Polynomial (n) 1Pnt 1Pnt 1Pnt nPnt, (n-1)Pnt, ..., 1Pnt
